# 专注陪伴工具（Companion-style Focus Tool）

一个本地优先的沉浸式专注应用，围绕「专注 → 休息 → 复盘」流程提供番茄钟、环境音、植物成长、习惯追踪、笔记和统计能力。
项目采用前后端同仓结构：前端为 React + Vite，服务端使用 Express 承载构建后的静态资源。

## 项目亮点

- **本地优先**：数据保存在浏览器本地，不依赖远程服务。
- **沉浸式专注体验**：番茄钟 + 环境音 + 场景化视觉联动。
- **正反馈机制**：植物成长和统计面板让专注成果可视化，顺便提供些情绪价值（bushi）。
- **学习与习惯协同**：笔记、习惯和待办功能和番茄钟提高学习或工作效率。

## 核心功能

- **番茄钟**：支持专注/休息状态切换，支持用户自定义番茄钟时长、休息时长和番茄钟轮数。
- **音效及音乐**：多音轨混音与预设场景音效，和本地音乐播放器。
- **植物成长系统**：用好感度和专注时长养大一株植物。
- **待办系统**：支持设置待办，并可设置标签和优先级。
- **习惯追踪**：按周查看完成情况（周日到周六）。
- **笔记系统**：支持笔记管理，并可拖动生成便利贴。
- **日历页**：支持日历功能，显示当月统计数据，还可以记日记。
- **数据统计**：学习时长、历史趋势与可视化展示。
- **个性化**：支持导入/导出存档，支持用户上传背景图片（尺寸裁切适应）。

## 项目结构

```text
.
├─ client/                 # 前端应用（Vite + React）
│  ├─ public/              # 静态资源
```

## 常用脚本说明

- `pnpm dev`：启动前端开发环境。
- `pnpm check`：TypeScript 类型检查（不产出文件）。
- `pnpm build`：构建前端 + 打包服务端。
- `pnpm start`：运行生产构建结果。
- `pnpm preview`：Vite 预览模式。
- `pnpm format`：格式化代码。
- `pnpm local:dev`：安装依赖后启动开发环境。
- `pnpm local:preview`：安装依赖后构建并启动生产服务。

此外，`scripts/` 目录中包含 Windows / macOS / Linux 的快捷脚本，可按需使用。

## 数据存储与隐私

当前版本默认使用浏览器 `localStorage` 存储本地数据，例如：

- 专注时长与相关统计
- 植物成长状态
- 笔记与便利贴状态
- 习惯完成记录
- 音效配置
- 本地音乐文件

默认不会主动上传到外部服务器。
前四项可随存档导出/导入。

## 常见问题（FAQ）

### 1）启动后没有自动打开浏览器怎么办？

某些运行环境（如精简 Linux 容器）可能缺少 `xdg-open`，会导致 `--open` 失败，但**不影响服务启动**。可手动访问终端输出的地址。

### 2）开发端口与预期不一致？

当默认端口被占用时，Vite 会自动切换可用端口，请以控制台输出为准。

### 3）生产启动页面异常？

请确认先执行了：

```bash
npm install
```

然后在项目根目录执行：

```bash
npm run dev
```
