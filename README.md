# 专注陪伴工具（Companion-style Focus Tool）

一个本地优先的沉浸式专注应用，围绕「专注 → 休息 → 复盘」流程提供番茄钟、环境音、植物成长、习惯追踪、笔记和统计能力。项目采用前后端同仓结构：前端为 React + Vite，服务端使用 Express 承载构建后的静态资源。

## 项目亮点

- **本地优先**：核心数据默认保存在浏览器本地，无需依赖远程服务即可使用。
- **沉浸式专注体验**：番茄钟 + 环境音 + 场景化视觉联动。
- **正反馈机制**：植物成长和统计面板让专注成果可视化。
- **学习与习惯协同**：笔记、习惯和专注时长互相补充。

## 核心功能

- **番茄钟**：支持专注/休息状态切换。
- **环境音模块**：多音轨混音与场景化播放。
- **植物成长系统**：专注进度驱动成长反馈。
- **笔记系统**：支持笔记管理和便利贴交互。
- **习惯追踪**：按周查看完成情况（周日到周六）。
- **数据统计**：学习时长、历史趋势与可视化展示。

## 快速开始

### 环境要求

- Node.js 22+
- 推荐使用 pnpm（项目 `packageManager` 已指定 pnpm）

### 安装依赖

```bash
pnpm install
```

> 如果你习惯 npm，也可以使用 `npm install`，但推荐与项目脚本保持一致使用 pnpm。

### 启动开发环境

```bash
pnpm dev
```

默认会启动 Vite 开发服务（命令中包含 `--host --open`）。实际端口请以终端输出为准。

### 类型检查

```bash
pnpm check
```

### 生产构建与启动

1. 构建：

```bash
pnpm build
```

2. 启动生产服务：

```bash
pnpm start
```

默认端口为 `3000`。你也可以通过环境变量覆盖：

```bash
PORT=8080 pnpm start
```

## 常用脚本说明

- `pnpm dev`：启动前端开发环境。
- `pnpm check`：TypeScript 类型检查（不产出文件）。
- `pnpm build`：构建前端 + 打包服务端。
- `pnpm start`：运行生产构建结果。
- `pnpm preview`：Vite 预览模式。
- `pnpm format`：格式化代码。
- `pnpm local:dev`：安装依赖后启动开发环境。
- `pnpm local:preview`：安装依赖后构建并启动生产服务。

此外，`scripts/` 目录中包含 Windows / macOS / Linux 的快捷脚本，可按需使用。

## 数据存储与隐私

当前版本默认使用浏览器 `localStorage` 存储本地数据，例如：

- 专注时长与相关统计
- 植物成长状态
- 笔记与便利贴状态
- 习惯完成记录
- 音效配置

默认不会主动上传到外部服务器。

## 常见问题（FAQ）

### 1）启动后没有自动打开浏览器怎么办？

某些运行环境（如精简 Linux 容器）可能缺少 `xdg-open`，会导致 `--open` 失败，但**不影响服务启动**。可手动访问终端输出的地址。

### 2）开发端口与预期不一致？

当默认端口被占用时，Vite 会自动切换可用端口，请以控制台输出为准。

### 3）生产启动页面异常？

请确认先执行了：

```bash
pnpm build
```

然后在项目根目录执行：

```bash
pnpm start
```
